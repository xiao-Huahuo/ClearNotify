一、自由结构解析与图谱生成
重点：政策文本结构差异极大，系统需要将任意格式的公文解析为可交互的知识图谱，生成 nodes / links / dynamic_payload 等图谱载荷，支持前端可视化展示与原文节点联动，而非输出固定字段卡片。
难点：大语言模型输出不稳定，可能夹带 Markdown、解释前缀或截断 JSON。系统采用"严格 JSON 约束输出 + 宽松修复 + Repair-Only 二次修复 + 失败回退"四级容错链路，并在模型只给出少量抽象节点时，基于原文行块、章节编号补出原文结构图层，保证即使解析不理想，前端仍能展示最小可用结果。

二、实时推理轨迹流式编排
重点：智能体的推理过程需要逐步可见，让用户看到每一步工具调用与思考过程，而非等待结束后一次性返回结果，从根本上解决 AI 黑盒问题。
难点：后端推理速度与前端渲染速度不匹配。若直接同步推送，工具步骤会快速闪过，用户根本看不清。系统通过 asyncio.Queue + to_thread 将推理线程与推送协程解耦，前端为每条轨迹设置最小可读时长消费队列；同时引入 trace_done 信号将"工具过程结束"与"正文开始输出"显式分离，防止答案流覆盖最后几步推理轨迹。

三、多模态文档解析与 OCR 容错
重点：系统需要统一处理 PDF、Word、Excel、图片、截图、URL 等多种格式输入，将不同来源的非结构化材料纳入同一解析链路，输出一致的图谱结果。
难点：图文混排 PDF 需要按页面阅读顺序混排文本块与图片块，不能简单地先提文字再提图片；内嵌大量图片的 DOCX 若逐张串行 OCR 会导致严重超时，需自动检测图片数量并切换为整文件抽取策略。OCR 链路还需兼顾有无本地 PaddleOCR 环境的两种部署场景，在本地模型不可用时无缝回退至 Kimi LLM OCR。

四、RAG 检索与幻觉抑制
重点：政策解析场景对准确性要求极高，错误的办理材料、截止日期或适用对象会直接误导用户。系统通过 RAG 机制以官方原文约束模型输出，从根本上抑制幻觉，而非依赖模型自身的"记忆"。
难点：检索质量直接决定生成质量。低相似度的噪声片段反而会干扰模型输出。系统采用 top_k + score_threshold 双阈筛选，同时合并个人知识与公共知识并去重，并将每条证据的相似度分数直接展示在智能体界面，使 RAG 的检索过程从"后台黑盒"变为"可观测能力"。
