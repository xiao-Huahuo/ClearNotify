• 路由约定

  这个项目把“页面路由”和“接口路由”分开集中管理了。

  - 前端页面路由统一写在 web/src/router/index.js:25。仓库规范也明确要求：页面路由和守卫放 src/router/index.js，API 路径
    常量和请求拦截器放 src/router/api_routes.js，不要分散到别处，doc/dev/开发规范.md:18。
  - 前端接口地址常量统一写在 web/src/router/api_routes.js:4，并且 axios 的 baseURL 固定是 /api，web/src/router/
    api_routes.js:86。所以前端代码里写的是 /auth/captcha、/user/me、/agent/run 这种“后端真实路径”，浏览器真正发出去会自
    动变成 /api/auth/captcha、/api/user/me、/api/agent/run。
  - 后端路由统一在 app/main.py:89 里通过 include_router(prefix=...) 挂载。也就是说，各 app/api/routes/*.py 文件只写相对
    路径，最终路径由 prefix + 子路由 组成。

  前端路由怎么规定

  前端是 Vue Router 的 history 模式，web/src/router/index.js:25。

  主要规则是：

  - / 直接重定向到 /showcase，web/src/router/index.js:32
  - 业务页有 /home、/history、/favorites、/profile、/settings、/search、/rewrite、/todo、/agent、/public-opinion-hall、/
    certified-analysis、/policy-publish-center、/policy-swipe，web/src/router/index.js:33
  - 展示页有 /showcase、/showcase/discovery、/showcase/screen，web/src/router/index.js:51
  - /admin 带 requiresAdmin，前置守卫会校验用户是否管理员，不是就跳到 /agent，web/src/router/index.js:45 web/src/router/
    index.js:57
  - login、register、discovery-home 这类路由带 meta.standalone，展示页带 meta.showcase，说明它们走不同布局，web/src/
    router/index.js:34

  另外，/agent 页面还会把会话 id 放在查询参数里，例如 /agent?conversation_id=123，用于恢复会话，web/src/views/
  Agent.vue:404。

  后端路由怎么规定

  后端实际没有 /api 这个前缀，真实接口从根开始挂：

  - /auth
  - /user
  - /login
  - /chat
  - /analysis
  - /settings
  - /upload
  - /news
  - /search
  - /todo
  - /favorite
  - /admin
  - /agent
  - /history
  - /showcase
  - /policy-documents
  - /opinions
  - /media 静态文件

  这些都在 app/main.py:87 app/main.py:90。

  以 agent 为例：

  - 路由前缀是 /agent，app/main.py:102
  - 子路由文件里定义了 /run、/conversations、/conversations/{id}/messages、/ws，app/api/routes/agent.py:154 app/api/
    routes/agent.py:178 app/api/routes/agent.py:226 app/api/routes/agent.py:244
  - 所以最终真实地址就是 /agent/run、/agent/conversations、/agent/conversations/{id}/messages、/agent/ws

  Docker 下 nginx 如何反向代理

  容器编排在 docker-compose.yml:11：

  - app 容器跑 FastAPI，监听 8080，docker-compose.yml:16
  - web 容器跑 nginx，对外暴露 80，docker-compose.yml:77

  nginx 配置在 web/nginx.conf:1，它做了 4 件事：

  - location /：直接服务前端静态文件，并用 try_files $uri $uri/ /index.html; 支持 Vue history 路由。所以你直接访问 /
    showcase/screen、/agent 不会 404，而是回退到 index.html 再交给前端路由处理，web/nginx.conf:6
  - location /api/：把所有 /api/* 转发到 http://app:8080/*，web/nginx.conf:49。这里因为 proxy_pass http://app:8080/; 末
    尾带 /，所以 /api/user/me 会被转成后端真实路径 /user/me
  - location /media/：把媒体文件代理到后端 /media/，web/nginx.conf:12
  - location /api/agent/ws：把前端 WebSocket 地址代理到后端真实 WebSocket /agent/ws，并打开升级头，web/nginx.conf:21

  实际链路是这样：

  - 页面请求：浏览器 GET /agent -> nginx 返回 index.html -> Vue Router 渲染 /agent
  - 普通接口：浏览器 GET /api/search/unified -> nginx -> app:8080/search/unified
  - 媒体资源：浏览器 GET /media/docs/a.pdf -> nginx -> app:8080/media/docs/a.pdf
  - WebSocket：前端连 /api/agent/ws?token=...，web/src/views/Agent.vue:472 -> nginx 代理到后端 /agent/ws，web/
    nginx.conf:21

  补一句，开发环境也用了同样思路，只不过不是 nginx，而是 Vite 代理：/api 和 /media 都会代理到 127.0.0.1:8080，并把 /api
  前缀去掉，web/vite.config.js:18。
